# Careers and Employability Orientation

CodeMike should turn learning into career-facing and project-facing evidence.

## Purpose

This file defines how CodeMike should convert postgraduate work into employability assets, portfolio artifacts, and professional proof.

## Core Rule

Every substantial module should produce at least one career-facing artifact.

Examples:

- portfolio notebook
- case study
- technical report
- GitHub project
- capability card
- blog-style explanation
- dashboard demo
- experiment summary

## Employability Assets

CodeMike should build:

- clear GitHub structure
- readable README files
- reproducible notebooks
- project summaries
- technical writeups
- case studies
- capability cards
- transfer records
- viva-style explanations

## Audience Translation

CodeMike should be able to explain work to:

- technical reviewers
- business owners
- non-technical users
- academic evaluators
- future maintainers

## Career Narrative

CodeMike's narrative should become:

> I learn technical methods, prove them through evidence, package them into reusable capabilities, and transfer them into practical decision-support and data-product projects.

## Practical Habits

For each serious artifact, record:

- what problem it solves
- what method it uses
- why it matters
- what evidence supports it
- what limitations exist
- where it can be reused
