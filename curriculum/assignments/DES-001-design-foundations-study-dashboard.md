## 1. Assignment Purpose

This assignment formalises CodeMike's first design-learning artifact.

The goal is not to create a pretty HTML page. The goal is to demonstrate that CodeMike can study UI/UX design foundations properly, compare multiple sources, generate notes, translate design principles into implementation criteria, and produce a reviewable artifact that can guide future interface work.

## 2. Background

The destination master browser currently works at a functional level. Before redesigning it, CodeMike must complete a design-foundation assignment.

The study dashboard must cover the design foundations needed to improve data-review HTML artifacts such as:

- destination master browser
- dataset browsers
- dashboard interfaces
- future PWA screens
- CodeMike learning artifacts

## 3. End Goal

At the end of the assignment, `docs/design-foundations.html` should function as a submitted university-style assignment.

It should be capable of being:

- reviewed
- marked against a rubric
- revised if below standard
- resubmitted if required
- promoted as a benchmark if excellent
- reused as a template for other projects if benchmark quality

## 4. Core Learning Requirement

Every deep-reading topic must use multiple sources.

Minimum standard per topic:

```text
at least 3 sources
at least 1 primary/high-authority source where possible
at least 1 practical/applied source
at least 1 cross-checking or alternative framing source
```

No topic is considered fully digested if it relies on one source only.

## 5. Curriculum Scope

The assignment must cover these 12 topics:

1. UI vs UX
2. What is UI design
3. UX design
4. Design thinking
5. Human-centered design
6. Gestalt principles
7. Fitts' law
8. Button states
9. Typography
10. Color theory
11. Web design / grid layout
12. Design systems

## 6. Required Output Per Topic

Each topic must include:

- topic definition
- source list
- source type classification
- notes from each source
- source comparison
- disagreements or emphasis differences
- CodeMike interpretation
- application to destination master browser
- anti-patterns to avoid
- implementation implications
- further reading suggestions
- checklist updates if applicable

## 7. Required Artifact Features

The final `docs/design-foundations.html` submission should include:

- clear assignment identity
- curriculum overview
- status by topic
- sources by topic
- source comparison notes
- CodeMike interpretation per topic
- browser-specific application per topic
- further reading suggestions
- implementation implications
- final synthesis
- readiness statement for browser redesign
