# Lab 01 — UI vs UX Browser Audit

## Lab objective

Execute the Topic 1 further-reading exercises against the current Destination Master Browser.

Target artifact:

```text
docs/destination-master-browser-v1.0.html
```

## Materials

- `design/foundations/ui-vs-ux-further-reading.md`
- `design/checklists/master-browser-design-checklist.md`
- `docs/destination-master-browser-v1.0.html`

## Lab steps

### Step 1 — UI inventory

List visible UI elements and answer:

- What UX task does this UI element support?
- What happens if it is removed?
- What risk does it reduce?

### Step 2 — UX journey map

Map the reviewer journey:

1. Understand dataset status
2. Search or filter records
3. Compare visible records
4. Inspect one record deeply
5. Identify uncertainty or enrichment need
6. Reset / change direction
7. Leave with correct trust level

### Step 3 — Heuristic mini-audit

Apply Nielsen's 10 usability heuristics to the browser.

Record for each heuristic:

- status: pass / partial / fail
- evidence
- severity
- recommendation

### Step 4 — UX Honeycomb score

Score the browser from 1 to 5 for:

- useful
- usable
- desirable
- findable
- accessible
- credible
- valuable
