# Reading Pack — Topic 04 Design thinking

## Required readings

| Source | Role |
|---|---|
| Stanford d.school — *Bootleg Bootcamp* (the design-thinking method cards) | Primary process source: the canonical five-stage workflow (Empathize / Define / Ideate / Prototype / Test) with concrete methods per stage. The most-cited diagram of design thinking. |
| IBM Design Thinking — *The Loop, Hills, Playbacks* (Enterprise Design Thinking framework) | Applied / team-scale framing: compresses the five stages into Observe / Reflect / Make and adds team-coordination devices (Hills as outcome statements, Playbacks as feedback gates). |
| Tim Brown — *Change by Design* (or the HBR essay "Design Thinking", 2008) | Conceptual / strategic foundation: design thinking as innovation discipline; the three-constraint frame (desirability / feasibility / viability). |
| Nielsen Norman Group — *Design Thinking 101* (and the connected NN/g articles) | Educational cross-check: the iterative/non-linear nature of the process; evidence-per-stage discipline; common failure modes. |

## Extension readings

| Source | Role |
|---|---|
| IDEO — *Field Guide to Human-Centered Design* (revisited from Topic 3 — different chapters this time) | The methods catalogue; bridges design thinking with HCD (Topic 5). |
| Jeanne Liedtka — *Design Thinking for the Greater Good* / HBR essays | Empirical research on design thinking as a *managerial* discipline; counterweight to the consultancy framings. |
| Kim Goodwin — *Designing for the Digital Age* (scenarios + personas chapters) | Operationalising Empathize / Define output into design-ready artifacts. |
| Don Norman — *Design Thinking: A Useful Myth* (2010 essay) + later reconsiderations | The most-cited skeptic. Useful for understanding the strongest critique: "design thinking" risks being a brand for what good designers already do, and risks under-respecting domain expertise. |
| GOV.UK Service Manual — *Discovery* phase | Public-sector framing of the Empathize / Define part of the loop; complements the d.school + IDEO consulting framings. |

## Reading questions

1. The five-stage canonical (d.school) and the three-activity Loop (IBM) describe the same shape. Where does each one's structure *help* and where does each *hide* something the other surfaces?
2. Tim Brown's three constraints (desirability / feasibility / viability) are widely-taught but under-operationalised: when can a candidate fail one and still ship? What does each source say (explicit or implicit) about that adjudication?
3. NN/g insists the process is iterative and non-linear, and that evidence is produced at every stage. Which jumps does NN/g treat as healthy (Test → Empathize, Prototype → Define)? Which jumps would be evidence-skipping failures?
4. Don Norman's *Useful Myth* critique: does it dismiss design thinking or sharpen it? How does each other source respond (implicitly) to Norman's objection that design thinking risks devaluing domain expertise?
5. Where does design thinking sit relative to Topic 3 (UX design — journey + criteria)? Use the four required sources to triangulate.
6. Where does design thinking sit relative to Topic 5 (HCD — standards-grade process)? Is it a subset, a competitor, or a different scale of the same discipline?
7. The CodeMike workspace is a single-person operation today. Which of the team-scale devices (IBM Hills, IBM Playbacks, d.school workshop facilitation patterns) translate; which don't?
8. Each source under-emphasises something. List one omission per source, and explain why the omission matters for a data-review tool like the Destination Master Browser.
