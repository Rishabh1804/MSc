# Reading Pack — Topic 03 UX design

## Required readings

| Source | Role |
|---|---|
| Don Norman — *The Design of Everyday Things* (introduction + chapter on user-centred design) | Primary conceptual foundation: UX as the discipline of fitting the design to the user, not the other way around |
| Nielsen Norman Group — Definition of User Experience + *10 Usability Heuristics* (revisited in journey context) | High-authority framing of UX scope and evaluation |
| IDEO — *The Field Guide to Human-Centered Design* (Methods section) | Practical UX-research methods, journey-mapping, and synthesis |
| GOV.UK Service Manual — *Learning about users and their needs* + *Map a user's journey through a service* | Public-service rigour on user-needs and journey-mapping. The strictest source on "user need vs user request vs solution-shape". |
| Interaction Design Foundation — *UX Design* (foundations course summary) | Educational cross-check on the discipline's scope and method |

## Extension readings

| Source | Role |
|---|---|
| Kim Goodwin — *Designing for the Digital Age* (personas + scenarios chapters) | Operationalising research into design artifacts |
| Indi Young — *Mental Models* | Aligning system structure to the user's existing mental model |
| Peter Morville — UX Honeycomb (revisited from Topic 1) | Quality facets as a journey-level audit lens |
| Jeff Patton — *User Story Mapping* | Story-shaped journey representation; complements GOV.UK journey maps |
| ISO 9241-210 (revisited via Topic 5 preview) — Human-centred design lifecycle | Standards-grade UX process; useful as a cross-check on activity completeness |

## Reading questions

1. How does each source define UX design, and where do the definitions converge and diverge?
2. Which source treats *user need* as strictly distinct from *user request*? How does that strictness change a designer's options?
3. Each source has a model of the UX process. Where do those models agree on the activities? Where do they differ on sequence or emphasis?
4. How is a *journey map* constructed in IDEO vs GOV.UK vs Norman's task-analysis framing? Which is most appropriate for the Destination Master Browser, and why?
5. What does each source say about evaluation? Where does Topic 1's Nielsen-heuristic audit fit in that picture?
6. Which sources discuss data trust or uncertainty signalling in user journeys? (Most do not — note this absence.)
7. How does each source bridge from research/modelling to design? What artifact is the bridge?
8. Where does each source's framing break down for an internal reviewer tool (vs a consumer product or public service)?

## Extraction target

The reading should produce:

- topic definition of UX design (with verbs)
- the four canonical UX activities (research / modelling / design / evaluation) with the artifact each produces
- source-by-source notes (what each teaches / what it omits / strongest takeaway)
- source comparison (agreement table + difference-by-emphasis table + per-source omissions)
- user-need framing (need vs request vs solution-shape) with three browser examples
- the seven-step reviewer journey for the Destination Master Browser with per-step *goal / cost / failure-mode / trust-check*
- the UX-acceptance-criterion form and three worked examples
- anti-patterns specific to UX work in data-review tools
- implementation implications for Browser v1.1 (the journey-level companion to Topic 2's component rule sheet)
- further reading and exercise tasks
- checklist updates for the master-browser checklist (Topic 3 gates)
