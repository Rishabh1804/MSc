# Quiz 05 — Human-centred design

## Instructions

Answer without looking at the answer key. Use examples from the Destination Master Browser wherever possible.

## Questions

1. Define human-centred design (HCD) as a discipline. State what it *is* and what it *is not*; distinguish it from design thinking and from UX design.

2. Name the four ISO 9241-210 activities and the artifact each one produces.

3. Name three of the six ISO 9241-210 principles, and explain how each translates to a single-person workspace like CodeMike's.

4. State the W3C accessibility / usability / inclusion triad. Give one example of a design that satisfies usability + accessibility but not inclusion.

5. Where does HCD sit relative to Topic 2 (UI), Topic 3 (UX), and Topic 4 (design thinking)? Use the canonical hierarchy.

6. Norman's *Human-Centered Design Considered Harmful?* argues HCD risks over-centring the individual user at the cost of systems thinking. State the argument and explain how ISO 9241-210's *context of use* activity addresses it.

7. Map four DES-001 artifacts (one each from Topics 2 / 3 / 4 + one from v1.1) to their ISO 9241-210 activity. Are any activities under-served?

8. The Destination Master Browser has three reviewer personas (first-time / power / accessibility-need). What does HCD say about whose context-of-use to centre when contexts diverge?

9. Give one anti-pattern specific to HCD in a single-person workspace.

10. Write an HCD self-audit gate that should be applied to every v1.2 backlog item before implementation. The gate should make a missing activity immediately visible.
