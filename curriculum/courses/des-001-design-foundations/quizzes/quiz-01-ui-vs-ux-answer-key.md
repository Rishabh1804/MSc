# Answer Key — Quiz 01 UI vs UX

## 1. Define UI

UI is the visible and interactive interface layer: controls, screens, layout, typography, colour, labels, components, and states.

## 2. Define UX

UX is the broader user experience of accomplishing a goal through the product, including task flow, trust, accessibility, error recovery, confidence, and usefulness.

## 3. Polished dashboard but poor UX

A dashboard can look attractive but fail if users cannot complete their task, cannot understand the data, or over-trust unverified records.

## 4. Browser UI elements and UX tasks

Examples:

- Search input: helps find a destination quickly.
- Filter controls: help narrow records by review criteria.
- Warning notice: helps prevent false trust in unverified records.

## 5. Data credibility as UX

The reviewer experience includes knowing whether records can be trusted. If credibility is unclear, the user may make unsafe decisions even if the UI is visually polished.

## 6. Attractive badges but no verification status

The problem is both UI and UX: the UI fails to display necessary status information, and the UX fails because the reviewer cannot judge trustworthiness.

## 7. Broadest source

NN/g / Don Norman gives the broadest scope because it defines UX as all aspects of the end-user interaction with the company, services, and products.

## 8. Design-decision gate

Example:

```text
Which reviewer task does this UI element improve, and which trust or workflow risk does it reduce?
```

## 9. Anti-pattern

Example:

```text
Adding decorative colours or cards that do not improve inspection, filtering, trust, or comprehension.
```

## 10. Accessibility and UX

Accessibility is part of UX because a product cannot provide a good experience if users cannot perceive, operate, understand, or reliably use it.
