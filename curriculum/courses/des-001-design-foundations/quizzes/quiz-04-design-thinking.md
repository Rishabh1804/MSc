# Quiz 04 — Design thinking

## Instructions

Answer without looking at the answer key. Use examples from the Destination Master Browser wherever possible.

## Questions

1. Define design thinking as a discipline in one or two sentences. State what it *is* and what it *is not*.

2. Name the five canonical d.school stages and the three IBM Loop activities. Map them onto each other.

3. State Tim Brown's three constraints and give one Destination Master Browser example for each.

4. NN/g says design thinking is iterative and non-linear. Name two specific jumps that count as *healthy* iteration and one jump that would be *evidence-skipping*.

5. List the evidence each stage must produce. A stage that produces no evidence has not been done — give one example for each stage of an evidence-free version of that stage that fails.

6. Don Norman's "Design Thinking: A Useful Myth" critique argues design thinking risks branding what good designers already do. State the critique and explain whether you think it sharpens or undermines the discipline.

7. Topic 3's GOV.UK user-need form and Topic 4's Define stage are doing related work. Explain the difference between a user-need statement and a point-of-view (POV) statement.

8. Give one anti-pattern specific to design thinking in a single-person workspace (where there is no team to facilitate, no users on hand, and the same person plays every role).

9. The execution plan pairs Topic 4 (do a loop) with Topic 3 (write criteria). Describe a browser-v1.2 scenario where you would do Topic 4 first, and a scenario where you would skip straight to Topic 3.

10. Write a design-decision gate question that should be asked before exiting Ideate (committing to one candidate for Prototype). The question should make a wrong answer immediately visible.
