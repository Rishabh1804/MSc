# Quiz 03 — UX design

## Instructions

Answer without looking at the answer key. Use examples from the Destination Master Browser wherever possible.

## Questions

1. Define UX design as a discipline in one or two sentences. The definition should include verbs and should distinguish UX design from UI design and from usability evaluation.

2. Name the four canonical UX activities and the artifact each produces.

3. A reviewer says, "I want a reset button on the filters". Classify this as a user need, a user request, or a solution-shaped need. Then rewrite it as a real user need.

4. State the seven steps of the Destination Master Browser reviewer journey. For each step, name the failure mode if the step is poorly supported.

5. A UX acceptance criterion has the form: *testable behaviour the UI must produce, expressed independently of the UI mechanism*. Write a UX acceptance criterion for the "narrow" step of the reviewer journey.

6. Topic 1's heuristic audit and Topic 3's acceptance criteria are both evaluation tools. What is the difference between them, and which is more reproducible across reviewers?

7. The GOV.UK Service Manual is the strictest source on user-needs. What discipline does it impose that the other Topic 3 sources do not?

8. Topic 2 ended at a component rule sheet. Topic 3 ends at an acceptance-criteria sheet. Why are both required before Browser v1.1 lands? Give one example of a v1.1 decision that needs both.

9. Give one Topic 3 anti-pattern specific to data-review tools.

10. Write a design-decision gate question that should be asked before adding a new step to the reviewer journey.
