# Quiz 02 — What is UI design

## Instructions

Answer without looking at the answer key. Use examples from the Destination Master Browser wherever possible.

## Questions

1. Define UI design as a discipline in one or two sentences. Make the definition include verbs.

2. Name the six element categories from the lecture and give one Destination Master Browser example for each.

3. List the nine standard component states. Which three are the most commonly missing in data-review tools and why?

4. Explain the difference between affordance, signifier, and feedback. Use a single component (e.g. a filter chip) to illustrate all three.

5. The browser currently shows records as cards. State two reviewer tasks for which a table would be a better container, and give one criterion that determines which container to show.

6. Give one example of an interface element in v1 whose signifier does not match its affordance. Describe the fix.

7. A reviewer applies four filters and the result set returns zero records. Describe the **empty state** the UI should show, including what content, what controls, and what tone.

8. Define "consistency" in UI design and explain why it reduces cognitive load. Give one inconsistency in v1 that a reviewer would have to learn.

9. Distinguish a drawer, a modal, and a full-page detail view. State one task in the Destination Master Browser that each is the best fit for.

10. Write one design-decision gate question that should be asked before adding a new UI component to the browser. The question should make a wrong answer immediately visible.
