## Behavioural-test summary

The walk-through exercises the full seven-step reviewer journey end-to-end:

1. **Arrive** — load + see banner + tool + dataset within 5s.
2. **Understand** — trust state is unambiguous (text + icon + colour).
3. **Narrow** — apply three filters across facets; observe perceptible count update; remove chips individually.
4. **Compare** — sort by trust; first comparison pass completed in ~100ms (well under the 60s budget).
5. **Inspect** — open drawer, navigate Prev/Next, close, scroll position preserved.
6. **Recover** — force empty state via aggressive search; recover in one click via Clear-all.
7. **Leave** — verify the trust state seen at list level matches the trust state at card level matches the trust state at drawer level.

The walk-through is reproducible: a second evaluator running the same script against the same build reaches the same 15 verdicts.

## Defects discovered during walk-through

**One test-logic defect found and fixed (in test only, not in implementation).**

U-NAR-2's first-pass assertion was inverted: it expected the removed chip's field to still show `has-value` and the still-applied chip's field to show default. The implementation was correct; the test reasoning about which chip the previous step had removed was wrong. Fixed by re-deriving the post-removal state and asserting against it correctly. Re-ran: pass.

No defects found in the v1.1 build itself.

## Lyra observations from PR #10 — closure check

The four polish items Lyra deferred to PR B are all closed:

| Observation | Status |
|---|---|
| `role="dialog"` + `aria-modal="false"` — ARIA convention | **Audited and justified.** Pattern is WAI-ARIA APG's non-modal-dialog. Added `aria-describedby` pointing at the drawer-trust line so screen readers announce trust state when the dialog gains focus. Comment in HTML documents the choice. |
| Focus restoration on drawer close | **Implemented.** `state.lastOpener` captures the row/card on first open; `closeDrawer` calls `opener.focus()` after the slide-out completes. Keyboard users resume at their journey position. |
| Result-count perceptibility animation | **Implemented.** `#resultCount.flash` adds a brief `--accent-soft` background flash on count change; reflow-forced so the transition restarts cleanly on every update. Closes U-NAR-1's "perceptible" requirement. |
| CSV URL override for offline testing | **Implemented.** `?csv=<url>` query-param overrides the default GitHub raw URL. Production loads unchanged; offline / staging URLs can be passed without code changes. |
| `console.warn` for unmapped trust values | **Implemented.** `_warnedTrustValues` deduplication set; first occurrence of any unknown `verification_status`/`planner_use_status` combination logs a warning. Defensive guard against future master-CSV taxonomy drift. |
