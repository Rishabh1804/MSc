# curriculum/

Academic work: assignments, courses, modules, and the thesis, all under one parent.

See `../charter/REPO_STRUCTURE.md` for layout rules. The existing course skeleton for `DES-001` (syllabus, weekly plan, labs, submissions, etc.) remains valid under `curriculum/courses/<course-code>/`.

## Contents

After Batch 4 this directory holds:

| Subdir | Source | What lives here |
|---|---|---|
| `assignments/` | Moved Batch 4 from `/assignments/` | Assignment briefs and rubrics. Currently: `DES-001-design-foundations-study-dashboard.md` and `rubrics/`. |
| `courses/` | Moved Batch 4 from `/courses/` | One subdirectory per course code. Currently: `des-001-design-foundations/` (syllabus, weekly plan, labs, feedback, learning log, grade reports). |
| `modules/` | Moved Batch 4 from `/modules/` | One subdirectory per taught module. Currently: `00-foundations`, `01-research-methods`, `02-big-data-analytics`, `03-data-mining-visualisation`, `04-group-project`, `05-parallel-programming`, `06-machine-learning-optimisation`, `07-optimisation`, `08-computational-intelligence`, `09-capstone`. |
| `thesis/` | Moved Batch 4 from `/thesis/` | Capstone thesis workspace (`README.md` only at present; populated as the capstone develops). |

## Authority

The course backbone in `../CLAUDE.md` (Core modules, Priority optional modules) is canonical for module selection. The module numbering scheme (`00-foundations` … `09-capstone`) is canonical for ordering. The course-code naming scheme (`des-001-design-foundations`) is canonical for course IDs.
