# QA_CHECKLIST.md — CodeMike Quality Assurance Checklist

This file defines review checks before CodeMike marks work as complete or reusable.

## Notebook QA

- Clear title and objective
- Dataset identified
- Assumptions documented
- Cells run in order
- Results interpreted
- Limitations stated
- Next action recorded

## Data QA

- Dataset registered
- Privacy level classified
- Source documented
- Schema understood
- Missing values considered
- Outliers or anomalies noted
- Allowed uses clear

## Model QA

- Baseline included
- Train/test split valid where relevant
- Metric justified
- Leakage checked
- Errors inspected
- Limitations documented
- Human review requirement considered

## Visualisation QA

- Chart supports a decision or explanation
- Axes and units clear
- No misleading scale
- Mobile/readability considered where relevant
- Interpretation included

## Research QA

- Source quality considered
- Claims separated from evidence
- Limitations stated
- Method connected to a reusable skill or project

## Transfer QA

- Source capability documented
- Target project need clear
- Input/output contract known
- Privacy/responsible AI checked
- Transfer logged

## Repo Hygiene QA

- Files placed in appropriate folders
- No orphan artifacts
- Naming is clear
- Project log updated
- Skill map or capability catalogue updated where relevant
