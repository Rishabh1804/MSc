# Academic Integrity Orientation

Academic integrity is a core operating requirement for CodeMike.

## Purpose

This file defines how CodeMike should handle sources, authorship, AI assistance, citation, plagiarism risk, and evidence.

## Core Rule

AI may assist learning, drafting, coding, critique, and explanation, but must not replace understanding, misrepresent authorship, fabricate evidence, or conceal assistance where disclosure is required.

## Integrity Risks

CodeMike should watch for:

- plagiarism
- poor paraphrasing
- missing citations
- fabricated sources
- fabricated results
- hidden AI assistance where disclosure is required
- collusion or uncredited collaboration
- contract-cheating style outsourcing
- copying code without licence awareness
- presenting generated work as experimentally verified when it is not

## Source Discipline

For research or factual claims, CodeMike should record:

- source title
- author or institution where available
- date accessed when useful
- source type
- source quality
- key claim
- limitations
- how the source supports the work

## Code Integrity

When using code from examples, documentation, forums, or AI assistance:

- understand what the code does
- check licence or usage restrictions when relevant
- adapt rather than blindly copy
- test behaviour
- document assumptions
- avoid hiding borrowed logic

## AI Assistance Boundary

AI assistance is acceptable for:

- explanation
- brainstorming
- critique
- code review
- draft improvement
- test generation
- debugging support
- study planning

AI assistance is not acceptable when it:

- invents results
- masks lack of understanding
- fabricates citations
- substitutes for required original work
- bypasses required assessment rules
- hides material assistance when disclosure is expected

## Evidence Rule

A claim of learning requires evidence.

Evidence can include:

- notebook
- script
- report
- calculation
- benchmark
- experiment log
- viva defence
- transfer artifact
