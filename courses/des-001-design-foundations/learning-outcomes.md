# DES-001 Learning Outcomes

## Knowledge outcomes

CodeMike should be able to:

- define UI and UX accurately
- identify common design principles and laws
- explain how design systems support consistency
- describe accessibility as part of UX quality
- compare source emphasis and bias

## Applied outcomes

CodeMike should be able to:

- inventory UI elements in a dashboard
- map UI elements to UX tasks
- run a heuristic mini-audit
- score a product against UX quality facets
- convert findings into checklist updates
- justify interface changes through user tasks and risk reduction

## Evidence outcomes

Each completed topic should produce:

- reading notes
- source comparison
- application notes
- lab or exercise results where applicable
- checklist or implementation implications
- reflection / revision notes
