# Pattern — Research Paper Summary Template

## Purpose

Use this pattern to turn a paper, report, or technical source into reusable understanding.

The goal is not to collect summaries. The goal is to extract methods, assumptions, limitations, and transferable capability.

## Header

```md
# Research Summary — <title>

Date reviewed:
Reviewer: CodeMike
Source type:
Linked module:
Linked capability:
Citation/link:
Status: proposed / reading / summarized / applied / retired
```

## 1. Source Quality

Record:

- author / institution
- publication venue or source
- publication date
- whether it is peer-reviewed
- whether it is official documentation, paper, blog, course note, or other source
- reason for trusting or treating cautiously

## 2. Research Question / Problem

What problem is the source addressing?

Write in plain language.

## 3. Motivation

Why does the problem matter?

Connect to:

- academic value
- practical project value
- data/product value
- CodeMike capability value

## 4. Method

Summarise the method in your own words.

Include:

- inputs
- outputs
- algorithm or process
- assumptions
- evaluation approach
- implementation notes

## 5. Key Claims

For each important claim:

```md
Claim:
Evidence given:
Strength of evidence:
Limitations:
```

## 6. Results

Summarise results without overclaiming.

Include:

- metrics used
- comparison baseline
- dataset/context
- whether results are generalisable

## 7. Limitations

Record limitations such as:

- sample size
- dataset bias
- unrealistic assumptions
- missing baselines
- weak evaluation
- narrow domain
- implementation complexity
- reproducibility concerns

## 8. What CodeMike Can Reuse

Extract reusable elements:

- method
- checklist
- algorithm
- metric
- evaluation design
- visualisation approach
- dataset idea
- experiment idea
- failure warning

## 9. Project Transfer Possibilities

Could this help:

- Planner
- Codex
- dashboards
- optimisation engines
- recommendation systems
- business analytics
- manufacturing/process analytics
- capstone/thesis

## 10. Follow-Up Questions

List questions that remain.

## 11. Summary in Five Lines

```md
Problem:
Method:
Main result:
Limitation:
Reusable value:
```

## Completion Criteria

- [ ] Source quality considered
- [ ] Method explained in own words
- [ ] Claims separated from evidence
- [ ] Limitations stated
- [ ] Reusable capability identified
- [ ] Transfer possibility recorded
