# patterns/

Reusable templates, checklists, and method patterns live here.

Patterns should make future work faster, safer, and more consistent.

## First Pattern Library

| Pattern | Purpose |
|---|---|
| `data-cleaning-checklist.md` | Prepare datasets safely and reproducibly before analysis or modelling |
| `eda-notebook-template.md` | Structure exploratory analysis notebooks as evidence artifacts |
| `research-paper-summary-template.md` | Convert papers and sources into reusable methods and limitations |
| `recommendation-scoring-pattern.md` | Rank options using criteria, weights, constraints, and explainable tradeoffs |
| `optimisation-problem-template.md` | Convert practical decisions into optimisation problems |
| `model-evaluation-template.md` | Evaluate models against baselines, error cases, and responsible AI risks |
| `dashboard-kpi-pattern.md` | Design decision-support dashboards with clear KPI definitions |
| `transfer-plan-template.md` | Move capability from MSc into target projects cleanly |

## Pattern Promotion Rule

Create a new pattern when a method is repeated, valuable, and likely to be reused.

Do not create patterns merely to fill the repo. Patterns should emerge from real work unless they are part of the foundation set.
