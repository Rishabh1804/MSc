# Capability — Project Transfer

## What this capability does

Moves a method, pattern, notebook, model, scoring system, report, or implementation from the MSc workspace into a target project cleanly and safely.

## Inputs

- Source capability
- Source evidence
- Target project need
- Transfer scope
- Input/output contract
- Risks and assumptions
- Acceptance criteria

## Outputs

- Transfer plan
- Adapted artifact
- Target-project implementation or documentation
- Transfer log entry
- Post-transfer review

## Method

Use `patterns/transfer-plan-template.md`.

Core sections:

1. Transfer summary
2. Target project need
3. Source artifact
4. Transfer scope
5. Adaptation needed
6. Risks
7. Acceptance criteria
8. Post-transfer review
