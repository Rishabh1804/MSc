# operations/

Live registers and logs. Files here change on the order of days as work is logged. Treat as append-mostly: editing an existing entry is allowed; deleting one requires a note in `DECISIONS.md`.

See `../charter/REPO_STRUCTURE.md` for this bucket's role in the overall layout, and `MIGRATION_PLAN.md` for the schedule by which existing root-level register docs move in.

## Contents

This directory currently holds:

- `MIGRATION_PLAN.md` — frozen target state for the restructure.
- `MIGRATION_LOG.md` — running record of batches as they commit.

The remaining 16 operations docs (`ARTIFACT_INDEX.md`, `CAPABILITIES.md`, `DATASETS.md`, `DECISIONS.md`, `EVIDENCE.md`, `EXPERIMENTS.md`, `FAILURE_LOG.md`, `LIBRARY.md`, `NEXT_ACTIONS.md`, `PORTFOLIO.md`, `PROJECT_LOG.md`, `RISK_REGISTER.md`, `ROADMAP.md`, `SKILL_MAP.md`, `TRANSFER_LOG.md`, `VIVA.md`) move in during Batch 3. `benchmarks/` and `trackers/` move in during Batch 6.
